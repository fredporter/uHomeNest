# v2.0.1 Local Runtime Alignment

This note records how `uHOME-server` fits into the `v2.0.1` platform spine
without changing ownership of the shared contracts.

## Contract Edges

- `uDOS-core` owns canonical workflow, binder, vault, and capability semantics
- `uDOS-shell` owns shared shell interaction patterns presented by sibling
  products
- `uDOS-wizard` owns provider and online transport routing
- `uHOME-server` owns the always-on local runtime, session execution, and
  nearby client surfaces
- `uHOME-empire` owns Google, HubSpot, and webhook-style sync beyond the local
  network

## Starter Runtime Role

For `v2.0.1`, `uHOME-server` is the public local runtime consumed by:

- `uHOME-client`
- `omd-mac-osx-app`
- `uHOME-app-android`
- `uDOS-wizard` local pairing flows

## Scope

This alignment note is additive only. It does not replace the active runtime or
the local work already in progress in this repo.
